# 编译部署手册

本文从零开始说明如何部署授权服务端、运行客户端、打包 Windows / macOS / Linux 客户端，以及发布前需要检查的事项。

## 1. 项目结构

```text
openyidalogin/
├── license-server/       # FastAPI 授权服务端和 Web 管理后台
├── electron-client/      # Electron 宜搭自助登录客户端
├── scripts/              # 密钥生成、卡密创建等辅助脚本
├── docker-compose.yml    # 服务端 Docker Compose 配置
├── README.md             # 快速说明
└── docs/编译部署手册.md   # 本文档
```

服务端负责卡密、设备绑定、JWT 签发和管理后台；客户端负责激活授权、打开宜搭登录页并保存 Cookie。

## 2. 服务端部署

### 2.1 准备服务器

推荐环境：

```text
Ubuntu / Debian
Docker
Docker Compose Plugin
```

确认 Docker 可用：

```bash
docker version
docker compose version
```

如果命令不存在，请先安装 Docker 和 Compose Plugin。

### 2.2 上传项目代码

把项目放到服务器，例如：

```bash
cd /opt
git clone <你的仓库地址> openyidalogin
cd openyidalogin
```

如果不是 Git 部署，也可以直接上传整个项目目录。

### 2.3 生成 RS256 密钥

上线前必须生成自己的密钥：

```bash
./scripts/generate_rs256_keys.sh
```

生成后会得到：

```text
license-server/keys/private.pem   # 服务端私钥，只能留在服务器
license-server/keys/public.pem    # 服务端公钥
electron-client/keys/public.pem   # 客户端内置公钥
```

注意：

- `private.pem` 不能发给客户。
- 重新生成密钥后，需要重新部署服务端并重新打包客户端。
- 已用旧公钥打包的客户端无法验证新私钥签发的授权 token。

### 2.4 配置服务端环境变量

复制示例配置：

```bash
cp license-server/.env.example license-server/.env
```

编辑配置：

```bash
nano license-server/.env
```

推荐内容：

```env
APP_NAME=License Server
DATABASE_URL=sqlite:///./data/license.db
JWT_ALGORITHM=RS256
PRIVATE_KEY_PATH=/app/keys/private.pem
PUBLIC_KEY_PATH=/app/keys/public.pem
ADMIN_USERNAME=admin
ADMIN_PASSWORD=请改成足够长的后台密码
ALLOW_INSECURE_DEFAULTS=false
```

说明：

- `ADMIN_PASSWORD` 不能使用 `admin123456`、`change-me` 等默认弱密码。
- `ALLOW_INSECURE_DEFAULTS=false` 适合生产环境。
- SQLite 数据库默认保存在 `license-server/data/license.db`。

### 2.5 启动服务端

构建并启动：

```bash
docker compose up -d --build
```

查看日志：

```bash
docker logs -f license-server
```

看到类似内容表示启动成功：

```text
Application startup complete
Uvicorn running on http://0.0.0.0:8000
```

检查健康接口：

```bash
curl http://127.0.0.1:8000/health
```

正常返回：

```json
{"ok": true}
```

### 2.6 访问管理后台

浏览器打开：

```text
http://服务器IP:8000/admin
```

使用 `.env` 中的账号密码登录：

```text
ADMIN_USERNAME / ADMIN_PASSWORD
```

后台可执行：

- 创建卡密
- 查看卡密
- 启用 / 禁用卡密
- 续期
- 查看绑定设备
- 解绑设备
- 查看激活和校验日志

### 2.7 常用服务端命令

查看容器状态：

```bash
docker compose ps
```

重启服务：

```bash
docker compose restart
```

停止服务：

```bash
docker compose down
```

重新构建：

```bash
docker compose up -d --build
```

查看最近日志：

```bash
docker logs --tail=200 license-server
```

## 3. HTTPS 和反向代理

生产环境建议使用 HTTPS。下面以 Nginx 为例。

### 3.1 Nginx 配置示例

```nginx
server {
    listen 80;
    server_name license.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name license.example.com;

    ssl_certificate /etc/letsencrypt/live/license.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/license.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

检查配置并重载：

```bash
nginx -t
systemctl reload nginx
```

### 3.2 客户端授权地址

如果服务端域名是：

```text
https://license.example.com
```

客户端授权地址必须写成：

```text
https://license.example.com/api/v1
```

不要漏掉 `/api/v1`。

## 4. 服务端本机开发运行

不使用 Docker 时，可以在本机直接运行 FastAPI。

```bash
cd openyidalogin
python3 -m venv .venv
. .venv/bin/activate
pip install -r license-server/requirements.txt
```

设置环境变量并启动：

```bash
export PYTHONPATH=license-server
export DATABASE_URL=sqlite:///./license-server/data/license.db
export PRIVATE_KEY_PATH="$PWD/license-server/keys/private.pem"
export PUBLIC_KEY_PATH="$PWD/license-server/keys/public.pem"
export ADMIN_PASSWORD="你的开发后台密码"

uvicorn app.main:app --host 127.0.0.1 --port 8000
```

访问：

```text
http://127.0.0.1:8000/admin
```

## 5. 客户端开发运行

### 5.1 安装 Node.js

建议使用 Node.js 22 LTS 或更新版本。

确认版本：

```bash
node -v
npm -v
```

### 5.2 安装客户端依赖

```bash
cd electron-client
npm install
```

### 5.3 启动开发版客户端

客户端需要知道授权服务地址。

Windows PowerShell：

```powershell
$env:AUTH_SERVER="https://license.example.com/api/v1"
$env:CLIENT_STORAGE_SECRET="开发用随机字符串"
npm start
```

Windows CMD：

```cmd
set AUTH_SERVER=https://license.example.com/api/v1
set CLIENT_STORAGE_SECRET=开发用随机字符串
npm start
```

macOS / Linux：

```bash
AUTH_SERVER=https://license.example.com/api/v1 CLIENT_STORAGE_SECRET=开发用随机字符串 npm start
```

本机联调时可以使用：

```text
http://127.0.0.1:8000/api/v1
```

### 5.4 激活和保存结果

客户端启动后：

1. 输入后台创建的卡密。
2. 激活成功后，右侧会打开宜搭登录页面。
3. 使用钉钉扫码登录。
4. 客户端检测到登录态后会自动保存 Cookie。

默认保存位置：

```text
桌面/yida-login-result/.cache/cookies-public.json
```

文件格式：

```json
{
  "cookies": [],
  "base_url": "https://www.aliwork.com"
}
```

## 6. 客户端打包配置

正式打包前必须配置授权地址和本地授权存储密钥。

编辑：

```text
electron-client/package.json
```

填写：

```json
"config": {
  "authServer": "https://license.example.com/api/v1",
  "storageSecret": "替换为随机长字符串"
}
```

说明：

- `authServer` 是客户端激活和在线校验接口地址。
- `storageSecret` 用于加密本地授权文件。
- 所有 `build:*` 命令都会先检查这两个配置。
- 也可以用 `AUTH_SERVER` 和 `CLIENT_STORAGE_SECRET` 环境变量临时覆盖。

生成随机字符串示例：

macOS / Linux：

```bash
openssl rand -hex 32
```

Windows PowerShell：

```powershell
[guid]::NewGuid().ToString("N") + [guid]::NewGuid().ToString("N")
```

## 7. 三平台客户端打包

最稳妥的方式是在对应系统上打对应平台包。

### 7.1 Windows 打包

在 Windows 上执行：

```powershell
cd electron-client
npm install
npm run build:win
```

输出目录：

```text
electron-client/dist/
```

默认产物：

```text
yida-self-login-版本号-x64.exe
```

建议：

- 正式分发前配置 Windows 代码签名证书。
- 未签名程序可能触发 SmartScreen 提示。

### 7.2 macOS 打包

在 macOS 上执行：

```bash
cd electron-client
npm install
npm run build:mac
```

默认产物：

```text
yida-self-login-版本号-x64.dmg
yida-self-login-版本号-x64.zip
yida-self-login-版本号-arm64.dmg
yida-self-login-版本号-arm64.zip
```

建议：

- 正式分发需要 Developer ID 证书。
- 面向普通用户分发时建议做 notarization。
- 没有证书时会使用 ad-hoc 签名，用户首次打开可能看到系统安全提示。

### 7.3 Linux 打包

在 Linux 上执行：

```bash
cd electron-client
npm install
npm run build:linux
```

默认产物：

```text
yida-self-login-版本号-x64.AppImage
yida-self-login-版本号-x64.deb
```

说明：

- AppImage 适合直接运行。
- deb 适合 Debian / Ubuntu 系发行版。

### 7.4 交叉打包限制

不建议完全依赖一台机器交叉打所有平台包。

常见限制：

- Apple Silicon macOS 打 Windows 包可能遇到 Wine 架构问题。
- Apple Silicon macOS 打 Linux AppImage 可能遇到 AppImage 工具架构问题。
- macOS 签名和 notarization 只能在 macOS 环境中完整处理。

推荐使用三台原生系统或 CI runner 分别构建。

## 8. GitHub Actions 自动构建

仓库包含工作流：

```text
.github/workflows/build-clients.yml
```

### 8.1 配置 Secrets

在 GitHub 仓库中进入：

```text
Settings -> Secrets and variables -> Actions -> New repository secret
```

添加：

```text
AUTH_SERVER=https://license.example.com/api/v1
CLIENT_STORAGE_SECRET=你的随机长字符串
```

### 8.2 手动触发构建

进入：

```text
Actions -> Build Electron Clients -> Run workflow
```

工作流会在：

```text
macos-latest
windows-latest
ubuntu-latest
```

分别构建客户端，并上传 artifacts。

### 8.3 tag 触发构建

推送 `v*` tag 会自动构建：

```bash
git tag v2.0.0
git push origin v2.0.0
```

## 9. 创建测试卡密

### 9.1 使用 Web 后台

1. 打开 `https://license.example.com/admin`。
2. 使用管理员账号登录。
3. 在“创建卡密”区域填写数量、有效期、设备数。
4. 点击创建并复制卡密。

### 9.2 使用脚本

脚本需要显式提供后台密码：

```bash
ADMIN_PASSWORD="你的后台密码" SERVER="https://license.example.com" ./scripts/create_license.sh
```

本机服务端联调：

```bash
ADMIN_PASSWORD="你的后台密码" SERVER="http://127.0.0.1:8000" ./scripts/create_license.sh
```

## 10. 发布前检查清单

发布前建议逐项确认：

```text
1. 已执行 scripts/generate_rs256_keys.sh 生成正式密钥
2. license-server/.env 已设置强后台密码
3. license-server/keys/private.pem 没有提交到 Git
4. electron-client/keys/public.pem 是最新公钥
5. electron-client/package.json 已填写 config.authServer
6. electron-client/package.json 已填写 config.storageSecret
7. 服务端 /health 正常
8. 后台能登录并创建卡密
9. 客户端能用测试卡密激活
10. 客户端能打开宜搭并生成 cookies-public.json
11. 三平台安装包均来自对应系统或 CI runner
```

## 11. 常见问题

### 11.1 服务端启动失败：后台密码不安全

现象：

```text
Refusing to start with an insecure default ADMIN_PASSWORD
```

解决：

```bash
nano license-server/.env
```

把 `ADMIN_PASSWORD` 改成强密码，然后重启：

```bash
docker compose up -d --build
```

### 11.2 后台密码改了但登录不上

服务端首次启动时会把管理员写入数据库。数据库已存在时，修改 `.env` 不会自动覆盖旧密码。

测试环境可以删除数据库：

```bash
docker compose down
rm -f license-server/data/license.db
docker compose up -d --build
```

生产环境不要直接删库，应增加管理员密码重置脚本后再操作。

### 11.3 客户端提示授权文件无效

常见原因：

- 服务端私钥和客户端公钥不匹配。
- 重新生成密钥后没有重新打包客户端。
- 本地旧授权文件仍在。

处理：

```bash
./scripts/generate_rs256_keys.sh
docker compose up -d --build
cd electron-client
npm run build:win
```

清理旧授权文件：

```text
Windows: C:\Users\你的用户名\AppData\Roaming\yida-electron-self-login\license.dat
macOS: ~/Library/Application Support/yida-electron-self-login/license.dat
Linux: ~/.config/yida-electron-self-login/license.dat
```

### 11.4 客户端请求 404

确认授权地址必须包含 `/api/v1`：

```text
https://license.example.com/api/v1
```

激活接口实际为：

```text
POST /api/v1/license/activate
```

### 11.5 打包时报配置不完整

现象：

```text
Client build configuration is incomplete.
```

解决：

编辑 `electron-client/package.json`，填写：

```json
"config": {
  "authServer": "https://license.example.com/api/v1",
  "storageSecret": "随机长字符串"
}
```

或者打包时传环境变量：

```bash
AUTH_SERVER=https://license.example.com/api/v1 CLIENT_STORAGE_SECRET=随机长字符串 npm run build:linux
```

### 11.6 macOS 提示无法打开应用

如果应用没有 Developer ID 签名和 notarization，macOS 可能提示无法验证开发者。

临时处理：

```text
系统设置 -> 隐私与安全性 -> 仍要打开
```

正式发布建议配置签名和 notarization。

### 11.7 Windows SmartScreen 提示

未签名或新签名程序可能被 SmartScreen 提示。

建议：

- 配置代码签名证书。
- 使用固定发布域名和稳定版本号。
- 不要频繁更换 appId / productName。

## 12. 安全注意事项

- `license-server/keys/private.pem` 只能保存在服务端。
- 客户端只能包含 `electron-client/keys/public.pem`。
- `license-server/.env` 不应提交到 Git。
- `license-server/data/` 不应提交到 Git。
- `config.storageSecret` 不应使用示例值。
- Electron 客户端无法做到绝对防破解，关键校验逻辑应尽量放在服务端。
